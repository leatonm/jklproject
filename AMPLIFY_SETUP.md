# AWS Amplify (Gen 2) + GitHub — JKL Platform

This app uses **Amplify Gen 2** (`ampx` + `amplify/` TypeScript backend). That is **different** from the older **Gen 1** CLI you may have installed globally:

| Tool | Role |
|------|------|
| `amplify` **v14** (global `npm i -g @aws-amplify/cli`) | Legacy Gen 1 — `amplify init`, `amplify push`. **Not** what this repo uses for backend definition. |
| `npx ampx` (from `@aws-amplify/backend-cli` in this project) | **Gen 2** — sandbox, pipeline deploy, matches `amplify/backend.ts`. |

Use **`npm run sandbox`** or **`npx ampx <command>`** for this codebase.

---

## 1. Local backend (sandbox)

1. Configure AWS credentials on your machine (access keys or SSO) for an IAM user/role that can deploy Amplify backends (see [AWS account setup](https://docs.amplify.aws/react/start/account-setup/); policy example: **AmplifyBackendDeployFullAccess** plus usual CloudFormation permissions).
2. From the project root:

   ```bash
   npm install
   npm run sandbox
   ```

3. Wait until deployment finishes. Root **`amplify_outputs.json`** is updated (gitignored). Your app reads it via `src/lib/configureAmplify.ts`.
4. In another terminal: **`npm run dev`**.

To stop sandbox: Ctrl+C in the sandbox terminal.

---

## 2. Connect GitHub → Amplify Hosting (fullstack)

1. Push this repo to **GitHub** (empty repo is fine; then add remote and push `main` or your default branch).
2. AWS Console → **Amplify** → **Create new app** → **Host web app** (fullstack / Gen 2 flow).
3. **Connect** your Git provider → authorize → choose the **repository** and **branch** (e.g. `main`).
4. Amplify should detect **`amplify.yml`** at the repo root. Build settings:
   - **Backend**: `npm ci` + `npx ampx pipeline-deploy` (already in `amplify.yml`).
   - **Frontend**: `npm run build`, artifact **`dist`** (Vite).
5. Create or select an **IAM service role** Amplify uses to deploy CloudFormation (the wizard can create one).
6. **Save and deploy**.

After a successful build, open the hosted URL. Download **amplify_outputs.json** from the branch’s **Deployed backend resources** tab if you need the same backend locally.

---

## 3. SPA routing (React Router)

If refreshing a deep link (e.g. `/roster`) returns **404** on the hosted URL, add a **rewrite** in Amplify Console → your app → **Hosting** → **Rewrites and redirects**:

- **Source**: `</^[^.]+$|\.(?!(css|gif|ico|jpg|js|png|txt|svg|woff|woff2|ttf|map|json|webp|jpeg|webmanifest)$)([^.]+$)/>`
- **Target**: `/index.html`
- **Type**: **200 (Rewrite)**

(Adjust the extension list if you add new static asset types.)

---

## 4. Environment / production

- **`amplify_outputs.json`**: Generated by sandbox or Hosting; **do not commit** (see `.gitignore`).
- **`.env.development`**: May set `VITE_DEV_AUTH_BYPASS=true` for local mock login. For real Cognito on the hosted site, rely on production build (or set `VITE_DEV_AUTH_BYPASS=false` in **Amplify Console → Environment variables** for the `main` branch if you add that variable to the frontend).

---

## 5. Useful commands

| Command | Purpose |
|---------|--------|
| `npm run sandbox` | Deploy backend to a **personal cloud sandbox**; refresh `amplify_outputs.json`. |
| `npm run ampx -- help` | List Gen 2 CLI commands. |
| `npm run build` | Production frontend build (needs valid `amplify_outputs.json` for typed config import). |

---

## 6. Two IAM users

Use **least privilege**: e.g. one human admin account and one **CI/deploy** user or role for Amplify Hosting, with policies scoped to this app/account as recommended by your org. Avoid sharing long-lived keys; prefer IAM Identity Center (SSO) where possible.

For questions specific to policy JSON, use the [Amplify Gen 2 IAM docs](https://docs.amplify.aws/react/start/account-setup/).
