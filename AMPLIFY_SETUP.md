# AWS Amplify (Gen 2) + GitHub ŌĆö JKL Platform

This app uses **Amplify Gen 2** (`ampx` + `amplify/` TypeScript backend). That is **different** from the older **Gen 1** CLI you may have installed globally:

| Tool | Role |
|------|------|
| `amplify` **v14** (global `npm i -g @aws-amplify/cli`) | Legacy Gen 1 ŌĆö `amplify init`, `amplify push`. **Not** what this repo uses for backend definition. |
| `npx ampx` (from `@aws-amplify/backend-cli` in this project) | **Gen 2** ŌĆö sandbox, pipeline deploy, matches `amplify/backend.ts`. |

Use **`npm run sandbox`** or **`npx ampx <command>`** for this codebase.

---

## 1. Local backend (sandbox)

If **`amplify_outputs.json`** contains placeholders like `REPLACE_USER_POOL_CLIENT_ID` or `REPLACE.appsync-api`, you only have the template from **`amplify_outputs.example.json`** (e.g. after `postinstall`). You need a **real** file from a successful deploy.

1. Configure AWS credentials on your machine (access keys or SSO) for an IAM user/role that can deploy Amplify backends (see [AWS account setup](https://docs.amplify.aws/react/start/account-setup/); policy example: **AmplifyBackendDeployFullAccess** plus usual CloudFormation permissions). If the CLI reports **Failed to load default AWS credentials**, run:

   ```bash
   npx ampx configure profile
   ```

   ŌĆ”or use **`aws configure`** / **`aws sso login`** with `--profile YourProfile` and pass **`npx ampx sandbox --once --profile YourProfile`**.

2. From the project root:

   ```bash
   npm install
   npm run sandbox
   ```

   For a **single** deploy (writes `amplify_outputs.json` then exits, no file watch):

   ```bash
   npm run sandbox:once
   ```

3. Wait until deployment finishes. Root **`amplify_outputs.json`** is updated (gitignored). Your app reads it via `src/lib/configureAmplify.ts`.
4. In another terminal: **`npm run dev`**.

To stop sandbox: Ctrl+C in the sandbox terminal (not needed if you used **`sandbox:once`**).

## 2. Connect GitHub ŌåÆ Amplify Hosting (fullstack)

1. Push this repo to **GitHub** (empty repo is fine; then add remote and push `main` or your default branch).
2. AWS Console ŌåÆ **Amplify** ŌåÆ **Create new app** ŌåÆ **Host web app** (fullstack / Gen 2 flow).
3. **Connect** your Git provider ŌåÆ authorize ŌåÆ choose the **repository** and **branch** (e.g. `main`).
4. Amplify should detect **`amplify.yml`** at the repo root. Build settings:
   - **Backend**: `npm ci --ignore-scripts` + `npx ampx pipeline-deploy`.
   - **Frontend**: `npm ci --ignore-scripts` + `npx ampx generate outputs` + `npm run build`, artifact **`dist`** (Vite).

### If deployment failed (common causes)

1. **Stale `amplify.yml` on GitHub** ŌĆö The repo must include the latest **`amplify.yml`** (with `--ignore-scripts` and **`generate outputs`** on the frontend). Commit and push, then **Redeploy** in Amplify.
2. **Backend phase** ŌĆö Open **Build logs** ŌåÆ **Backend** tab. Typical issues: IAM service role missing **CloudFormation / CDK** permissions, or the account/region needs **[CDK bootstrap](https://docs.aws.amazon.com/cdk/v2/guide/bootstrapping.html)** for Amplify Gen 2.
3. **Frontend phase** ŌĆö If **`generate outputs`** fails, the backend for that branch may not have finished deploying; fix backend errors first.
4. **Node version** ŌĆö In Amplify Console ŌåÆ **Build settings** ŌåÆ **Build image settings**, set **Node.js** to **20.x** if the log shows engine or syntax errors.
5. Create or select an **IAM service role** Amplify uses to deploy CloudFormation (the wizard can create one).
6. **Save and deploy**.

After a successful build, open the hosted URL. Download **amplify_outputs.json** from the branchŌĆÖs **Deployed backend resources** tab if you need the same backend locally.

## 5. SPA routing (React Router)

If refreshing a deep link (e.g. `/roster`) returns **404** on the hosted URL, add a **rewrite** in Amplify Console ŌåÆ your app ŌåÆ **Hosting** ŌåÆ **Rewrites and redirects**:

- **Source**: `</^[^.]+$|\.(?!(css|gif|ico|jpg|js|png|txt|svg|woff|woff2|ttf|map|json|webp|jpeg|webmanifest)$)([^.]+$)/>`
- **Target**: `/index.html`
- **Type**: **200 (Rewrite)**

(Adjust the extension list if you add new static asset types.)

## 6. Environment / production

- **`amplify_outputs.json`**: Generated by sandbox or Hosting; **do not commit** (see `.gitignore`).
- **`.env.development`**: May set `VITE_DEV_AUTH_BYPASS=true` for local mock login. For real Cognito on the hosted site, rely on production build (or set `VITE_DEV_AUTH_BYPASS=false` in **Amplify Console ŌåÆ Environment variables** for the `main` branch if you add that variable to the frontend).

## 6. Two IAM users

Use **least privilege**: e.g. one human admin account and one **CI/deploy** user or role for Amplify Hosting, with policies scoped to this app/account as recommended by your org. Avoid sharing long-lived keys; prefer IAM Identity Center (SSO) where possible.

For questions specific to policy JSON, use the [Amplify Gen 2 IAM docs](https://docs.amplify.aws/react/start/account-setup/).
